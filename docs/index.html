Plain Markdown in Git · gelesen bei Session-Start

Plain markdown in git your AI agent reads at session start — so context compounds instead of restarting.

Plain Markdown in Git, das dein KI-Agent bei Session-Start liest — damit Kontext wächst, statt jedes Mal neu zu starten.

Stop re-explaining your repos, your clients, and your conventions every session. open-bridge is plain markdown and YAML in git that your coding agent reads at the start of every session — so it already knows your world, independent of which model or frontend you run. No database, no SaaS, nothing to host.

Hör auf, deinem Agenten in jeder Session deine Repos, Kunden und Konventionen neu zu erklären. open-bridge ist Plain-Markdown und YAML in Git, das dein Coding-Agent zu Beginn jeder Session liest — damit er deine Welt schon kennt, unabhängig davon, welches Modell oder Frontend du nutzt. Keine Datenbank, kein SaaS, nichts zu hosten.

Context BoosterContext Booster

An independent context layer beside your model and your tool — coined "Context Booster" at BKS-Lab.

Eine modell- und frontend-unabhängige „dritte Seite" — als „Context Booster" geprägt bei BKS-Lab.

You work with whatever model and whatever frontend you like. open-bridge layers in persistent context about your world — your repos and clients, your machines, what you worked on yesterday — so the agent stops asking "GitHub or Jira?" and stops loading the wrong client. The boost comes from a third side that doesn't care what you run, because the substrate is plain text every runtime can read.

Du arbeitest mit dem Modell und dem Frontend, das du willst. open-bridge legt persistenten Kontext über deine Welt darüber — deine Repos und Kunden, deine Maschinen, woran du gestern gearbeitet hast — damit der Agent aufhört zu fragen „GitHub oder Jira?" und aufhört, den falschen Kunden zu laden. Der Boost kommt von einer dritten Seite, der egal ist, was du nutzt — weil das Substrat Plain Text ist, den jede Runtime lesen kann.

$ good morning, briefing → board · today's git activity · what's stale → already knows: which repos, which client, last week

Chaos TamerChaos Tamer

Structuring unstructured AI dialogues — coined at BKS-Lab (originally „KI-Chaos-Bändiger").

„Strukturierung unstrukturierter KI-Dialoge" — geprägt bei BKS-Lab als „KI-Chaos-Bändiger".

open-bridge ships ready-made structure you don't have to invent — and helps you keep it. It holds a structure (a board, a log, per-task status backlog → doing → review → done over tasks/ and streams/), files away knowledge as you work — decisions, findings and status land in plain text the agent reads back — and keeps the work from drifting into another My-Documents pile. Unstructured agent sessions become a persistent, filed, compounding record.

open-bridge liefert fertige Struktur mit, die du nicht erfinden musst — und hilft dir, sie zu halten. Es hält eine Struktur (ein Board, ein Log, Per-Task-Status backlog → doing → review → done über tasks/ und streams/), legt Wissen ab, während du arbeitest — Entscheidungen, Erkenntnisse und Status landen als Plain Text, den der Agent später wieder liest — und bewahrt die Arbeit davor, in den nächsten My-Documents-Haufen abzudriften. Aus unstrukturierten Agenten-Sessions wird ein persistentes, abgelegtes, wachsendes Protokoll.

work/board.md Doing (2) · bigcorp-checkout · startupxyz-handover Streams (1) · infra-hardening Done this month (7)

Bet / Roadmap Workspace separation as the default — keep each engagement's facts apart, and "if you can't place it into your known world-models, ask" instead of guessing. This is the intended design stance, agreed in principle, but not yet built into open-bridge. We label it honestly below. Workspace-Trennung als Default — die Fakten jeder Beauftragung getrennt halten, und „wenn du es nicht in deine bekannten Weltmodelle einordnen kannst, dann frage" statt zu raten. Das ist die beabsichtigte Design-Haltung, im Prinzip vereinbart, aber noch nicht in open-bridge eingebaut. Wir kennzeichnen das unten ehrlich.

Drei Befehle.
