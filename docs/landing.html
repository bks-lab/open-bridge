Plain Markdown in Git · gelesen bei Session-Start

Plain markdown in git your AI agent reads at session start — so context compounds instead of restarting.

Plain Markdown in Git, das dein KI-Agent bei Session-Start liest — damit Kontext wächst, statt jedes Mal neu zu starten.

open-bridge is a git repo of markdown and YAML — no database, no SaaS, nothing to host. Your coding agent reads it at the start of every session, so it already knows your repos, your clients, and how you work — independent of which model or frontend you run.

open-bridge ist ein Git-Repo aus Markdown und YAML — keine Datenbank, kein SaaS, nichts zu hosten. Dein Coding-Agent liest es zu Beginn jeder Session und kennt damit deine Repos, deine Kunden und deine Arbeitsweise — unabhängig davon, welches Modell oder Frontend du nutzt.

What the substrate enables

Was das Substrat ermöglicht

Two properties, named at BKS-Lab.

Zwei Eigenschaften, bei BKS-Lab benannt.

Both terms were coined by the team at BKS-Lab. The English view translates them plainly; the German view carries them verbatim.

Beide Begriffe wurden im Team bei BKS-Lab geprägt — hier in Originalform.

Context BoosterContext Booster

An independent context layer beside your model and your tool — coined "Context Booster" at BKS-Lab.

Eine modell- und frontend-unabhängige „dritte Seite" — geprägt bei BKS-Lab.

You work with whatever model and whatever frontend you like. open-bridge layers in persistent context about your world — your repos and clients, your machines, what you worked on yesterday — so the agent stops asking "GitHub or Jira?" and stops loading the wrong client. The boost comes from a third side that doesn't care what you run, because the substrate is plain text every runtime can read.

Du arbeitest mit dem Modell und dem Frontend, das du willst. open-bridge legt persistenten Kontext über deine Welt darüber — deine Repos und Kunden, deine Maschinen, woran du gestern gearbeitet hast — damit der Agent aufhört zu fragen „GitHub oder Jira?" und aufhört, den falschen Kunden zu laden. „Ein System, das mir von dritter Seite, unabhängig davon was ich gerade mache, einen Kontext gibt, der in jedem Fall hilfreich ist."

$ good morning, briefing → board · today's git activity · what's stale → already knows: which repos, which client, last week

KI-Chaos-Bändiger (AI-chaos tamer)KI-Chaos-Bändiger

Structuring unstructured AI dialogues — coined at BKS-Lab.

„Strukturierung von unstrukturierten KI-Dialogen" — geprägt bei BKS-Lab.

open-bridge ships ready-made example structures, so you don't invent your own filing chaos. The Task-Management system — a board, a log, and per-task status (backlog → doing → review → done) over tasks/ and streams/ — is the concrete instance: unstructured agent sessions become a persistent, filed, compounding record.

open-bridge liefert fertige Beispielstrukturen mit, damit du nicht dein eigenes Ablage-Chaos erfindest. Das Task-Management — ein Board, ein Log und Per-Task-Status (backlog → doing → review → done) über tasks/ und streams/ — ist die konkrete Ausprägung: aus unstrukturierten Agenten-Sessions wird ein persistentes, abgelegtes, wachsendes Protokoll.

work/board.md Doing (2) · bigcorp-checkout · startupxyz-handover Streams (1) · infra-hardening Done this month (7)

Bet / Roadmap Workspace separation as the default — keep each engagement's facts apart, and "if you can't place it into your known world-models, ask" instead of guessing. This is the intended design stance, agreed in principle, but not yet built into open-bridge. We label it honestly below. Workspace-Trennung als Default — die Fakten jeder Beauftragung getrennt halten, und „wenn du es nicht in deine bekannten Weltmodelle einordnen kannst, dann frage" statt zu raten. Das ist die beabsichtigte Design-Haltung, im Prinzip vereinbart, aber noch nicht in open-bridge eingebaut. Wir kennzeichnen das unten ehrlich.
